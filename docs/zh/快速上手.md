# 快速上手

本指南将帮助您快速了解 QuickDeck 的基本使用方法。

## 前提条件

- 已部署 QuickDeck 系统（参考 [部署说明](./部署说明.md)）
- 拥有有效的用户账号（管理员创建或自行注册）

## 第一步：登录系统

1. 打开浏览器，访问 QuickDeck 前端地址（默认：`http://localhost`）
2. 在登录页面输入您的用户名和密码
3. 点击"登录"按钮

## 第二步：创建项目

项目是工具的组织单位，所有工具都必须属于某个项目。

1. 登录后，点击左侧导航栏的"项目管理"
2. 点击"新建项目"按钮
3. 填写项目信息：
   - **项目名称**：输入项目名称（必填）
   - **项目描述**：输入项目描述（可选）
   - **项目ID**：系统自动生成，也可手动指定（可选）
4. 点击"确定"创建项目

创建成功后，项目会出现在项目列表中。

## 第三步：选择项目

在顶部导航栏的项目选择器中，选择您刚创建的项目。只有选择了项目后，才能创建和管理该项目的工具。

## 第四步：创建第一个工具

1. 点击左侧导航栏的"工具管理"
2. 点击"新建工具"按钮
3. 填写工具基本信息：
   - **工具名称**：例如"Hello World"
   - **工具路径**：例如"hello/world"（用于 URL 路径）
   - **工具描述**：描述工具的用途（可选）

4. 配置工作流（可选）：
   - 如果暂时不需要配置工作流，可以直接保存工具
   - 工作流配置包括：运行步骤、输入参数、超时设置等

5. 点击"保存"按钮

## 第五步：配置工作流（可选）

如果您想创建一个可执行的工具，需要配置工作流：

### 5.1 添加运行步骤

1. 在工具编辑页面，切换到"运行步骤"标签页
2. 点击"添加步骤"按钮
3. 选择步骤类型（例如：Bash 命令）
4. 配置步骤内容（例如：输入 `echo "Hello, QuickDeck!"`）
5. 点击"确定"保存步骤

### 5.2 添加输入参数（可选）

1. 切换到"输入参数"标签页
2. 点击"添加参数"按钮
3. 选择参数类型（文本、数字、日期等）
4. 填写参数信息：
   - **参数名称**：用于在步骤中引用
   - **显示名称**：在界面上显示的名称
   - **是否必填**：是否必须提供该参数
   - **默认值**：参数的默认值（可选）

### 5.3 配置其他选项

- **超时时间**：设置工具执行的超时时间（秒）
- **重试次数**：设置失败后的重试次数
- **定时执行**：配置定时执行规则（可选）

## 第六步：执行工具

1. 在工具列表中，找到您创建的工具
2. 点击工具右侧的"执行"按钮
3. 如果工具有输入参数，填写参数值
4. 点击"执行"按钮
5. 等待执行完成，查看执行结果

## 第七步：查看执行历史

1. 点击左侧导航栏的"执行历史"
2. 查看所有工具的执行记录
3. 点击某条记录可以查看详细的执行结果和日志

## 示例：创建一个简单的 Hello World 工具

让我们创建一个最简单的工具作为示例：

### 1. 创建工具

- **工具名称**：Hello World
- **工具路径**：hello/world
- **工具描述**：一个简单的 Hello World 工具

### 2. 添加运行步骤

- **步骤类型**：Bash 命令
- **命令内容**：`echo "Hello, QuickDeck!"`

### 3. 保存并执行

保存工具后，点击"执行"按钮，您应该能看到输出：`Hello, QuickDeck!`

## 提示

- **项目选择**：记住在创建工具前先选择项目
- **工具路径**：工具路径应该是唯一的，建议使用层级结构（如：`team/tool-name`）
- **测试工具**：在正式使用前，建议先创建一个简单的测试工具验证功能
- **查看日志**：如果工具执行失败，查看执行日志可以帮助您定位问题
